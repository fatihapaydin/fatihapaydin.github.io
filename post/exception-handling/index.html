Exception Handling in Asp.Net Core

09-04-2019 598 words 3 mins read

Exception handling and giving friendly messages to our users are basics of application development. How can make exception handling in asp.net core? In fact Asp.Net Core template Startup class comes basic structure as below:

    app.UseHsts();
}

If our app are working in development environment, we will take detailed information about any throwing exceptions. In production environment, showing code exception to user is obviously a security gap then we must log the exception and give our users some informational messages concerning thrown exception.

Tip

Accessing to Home/Error action from any exceptional code block, we should not use any http attribute (HttpGet etc) with error action.

public IActionResult Status(int id)
{
    //You should add Home/NotFound or Home/ServerError view page to your project
    switch (id)
    {
        case 404:
            return View("NotFound");

        case 500:
            return View("ServerError");

        default:
            return View();
    }
}
